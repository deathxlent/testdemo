# 电子书管理器 ver1 实用手册

## 一、应用概述

电子书管理器是一个跨平台的 GUI 电子书管理工具，支持 Windows、Linux 和 Mac 系统。基于 PyQt6 开发，使用 SQLite 数据库存储书籍信息，能够有效管理个人电子书库。

## 二、功能特性

### 2.1 核心功能

| 功能 | 说明 |
|------|------|
| 📚 书架模式 | 以表格列表形式展示所有电子书，包含封面缩略图 |
| 🔍 搜索功能 | 支持对标题、副标题、作者、分类、物理位置、扩展名、ISBN 进行模糊查询 |
| 🔄 排序功能 | 点击列头可按该列升序/降序排序 |
| 📥 导入功能 | 支持拖拽导入、文件选择、文件夹选择，自动跳过已存在的文件 |
| 📖 格式支持 | EPUB、MOBI、AZW3、AZW |
| ✏️ 详情管理 | 支持查看和修改所有字段信息，支持删除书籍记录 |
| 🚀 快速打开 | 双击列表项用系统默认应用打开电子书 |
| 🖼️ 封面显示 | 自动提取并显示 EPUB 封面 |
| 🌐 豆瓣解析 | 从豆瓣获取书籍详细信息（标题、作者、ISBN、评分、简介、封面等） |
| ✅ 批量操作 | 支持批量选择、批量解析豆瓣信息、批量删除 |

### 2.2 支持的书籍字段

| 字段 | 类型 | 说明 |
|------|------|------|

| updated_at | TIMESTAMP | 更新时间 |

## 三、安装和运行

### 3.1 安装依赖

```bash
pip install -r requirements.txt
```

**依赖包说明：**
- `PyQt6>=6.4.0` - GUI 框架
- `ebooklib>=0.18` - EPUB 文件解析
- `Pillow>=10.0.0` - 图像处理
- `beautifulsoup4>=4.12.0` - HTML 解析
- `lxml>=4.9.0` - XML/HTML 解析库
- `kindleunpack>=0.99.0` - MOBI/AZW3 文件解析

### 3.2 运行程序

```bash
python main.py
```

首次运行会自动创建：
- `ebook_manager.db` - SQLite 数据库文件
- `covers/` - 封面图片存储目录
- `config.json` - 配置文件（存储豆瓣 Cookie）

## 四、使用说明

### 4.1 导入电子书

**方式一：拖拽导入**
1. 点击菜单 `文件` → `导入电子书`
2. 将文件或文件夹直接拖拽到虚线区域

**方式二：按钮选择**
1. 点击菜单 `文件` → `导入电子书`
2. 点击 "选择文件" 或 "选择文件夹" 按钮
3. 点击 "开始导入" 按钮

**导入特性：**
- 自动递归扫描目录
- 自动跳过已存在的文件（通过物理路径去重）
- 后台线程导入，不阻塞 UI
- 显示实时导入进度

### 4.2 查看和编辑书籍信息

1. **单击** 列表中的任意位置打开详情窗口
2. 在详情窗口中：
   - 点击 "修改" 按钮进入编辑模式
   - 修改完成后点击 "保存"
   - 点击 "删除" 按钮删除该书籍记录（不会删除源文件）
   - 点击 "📖 打开书籍" 用系统默认应用打开

### 4.3 打开电子书

- **双击** 列表中的任意位置，将使用系统默认应用打开该电子书
- 或在详情窗口点击 "📖 打开书籍" 按钮
- 或在列表操作列点击 "📖 打开" 按钮

### 4.4 搜索书籍

- 在顶部搜索框输入关键词，将实时过滤匹配的书籍
- 支持搜索字段：标题、副标题、作者、分类、物理位置、扩展名、ISBN

### 4.5 排序

- 点击任意列标题，将按该列排序
- 再次点击同一列标题切换升序/降序
- 可排序列：标题、副标题、作者、分类、文件大小、物理位置、扩展名、ISBN、评分

### 4.6 批量操作

1. 点击左侧复选框选择书籍（或点击 "☑️ 全选" 按钮全选）
2. **批量解析豆瓣**：点击 "🔍 批量解析豆瓣" 按钮，可同时解析多本书的豆瓣信息
3. **批量删除**：点击 "🗑️ 批量删除选中" 按钮，可同时删除多条记录

## 五、豆瓣信息解析

### 5.1 功能说明

应用集成了豆瓣书籍信息解析功能，可以自动从豆瓣获取书籍的详细信息。

### 5.2 配置 Cookie

由于豆瓣有反爬虫机制，需要配置 Cookie 才能正常解析：

1. 在浏览器中登录 [豆瓣](https://www.douban.com)
2. 打开浏览器开发者工具（F12），找到网络请求
3. 复制任意请求中的完整 Cookie 值
4. 点击菜单 `设置` → `豆瓣配置`
5. 将 Cookie 粘贴到输入框中，点击 "保存"

### 5.3 单本解析

1. 打开书籍详情窗口
2. 点击 "🔍 从豆瓣解析" 按钮
3. 确认后开始解析，解析完成后自动更新书籍信息

### 5.4 批量解析

1. 在主界面勾选需要解析的书籍
2. 点击 "🔍 批量解析豆瓣" 按钮
3. 确认后开始批量解析

**注意事项：**
- 每次解析间隔至少 7 秒，避免触发豆瓣反爬机制
- 解析在后台队列中进行，不影响正常使用
- 解析状态会实时显示在封面列（⏳解析中/✓成功/✗失败）

### 5.5 豆瓣解析 Bug 说明

**当前版本存在的问题：**

`ver1/douban_parser.py` 中的豆瓣解析功能可能会失败，主要原因：

1. **豆瓣页面结构变化**：豆瓣页面的 HTML 结构可能已更新，导致正则表达式匹配失败
2. **反爬虫机制加强**：即使配置了 Cookie，也可能被豆瓣拦截
3. **搜索 API 限制**：`subject_suggest` API 可能返回空结果或被限制

**解析流程：**
1. 调用搜索 API：`https://book.douban.com/j/subject_suggest?q={关键词}`
2. 获取第一个搜索结果的 subject_id
3. 请求详情页面：`https://book.douban.com/subject/{subject_id}/`
4. 使用正则表达式解析 HTML 提取信息
5. 下载封面图片到本地

**建议的解决方案：**
- 参考 `fetch_info` 目录下的实现，该版本使用了更稳定的解析方式
- 使用 `requests` 库代替 `urllib`，提供更好的错误处理
- 使用 `lxml` + XPath 代替正则表达式进行 HTML 解析
- 增加更多的 User-Agent 轮换机制

## 七、常见问题

### Q1: 导入电子书后封面不显示？
A: MOBI 和 AZW3 格式的封面提取支持有限，建议使用 EPUB 格式。可以通过豆瓣解析功能获取封面。

### Q2: 豆瓣解析总是失败？
A: 请检查：
1. 是否已正确配置 Cookie
2. Cookie 是否过期（需要重新登录豆瓣获取）
3. 网络是否正常
4. 可以尝试使用 `fetch_info` 目录下的工具获取信息

### Q3: 删除书籍会删除源文件吗？
A: 不会。删除操作只会删除数据库中的记录，不会删除本地的电子书文件。

### Q4: 数据库文件在哪里？
A: 数据库文件 `ebook_manager.db` 位于程序运行目录下，可以使用 SQLite 工具直接打开查看。

### Q5: 如何备份数据？
A: 直接复制 `ebook_manager.db` 文件和 `covers/` 目录即可完成备份。

## 八、技术架构

### 8.1 技术栈
- **GUI 框架**: PyQt6
- **数据库**: SQLite3
- **EPUB 解析**: ebooklib
- **图像处理**: Pillow
- **HTML 解析**: BeautifulSoup4 + lxml

### 8.2 核心模块说明

#### database.py - 数据库操作
- 使用 SQLite3 存储数据
- 支持自动迁移（新增字段自动添加）
- 提供完整的 CRUD 操作

#### ebook_parser.py - 电子书解析
- **EPUB 解析**：使用 ebooklib 提取元数据（标题、作者、ISBN 等），多种策略提取封面
- **MOBI/AZW3 解析**：支持 kindleunpack 库解析，或基础二进制解析
- 目录递归扫描，自动识别支持的格式

#### douban_parser.py - 豆瓣解析
- 线程队列实现异步解析
- 自动限速（7秒间隔）
- 支持搜索和详情解析
- 自动下载封面图片

### 8.3 豆瓣解析工作流

```
用户点击解析
    ↓
加入解析队列
    ↓
工作线程取出任务
    ↓
等待限速（距离上次解析至少7秒）
    ↓
调用豆瓣搜索 API
    ↓
获取 subject_id
    ↓
请求书籍详情页面
    ↓
正则表达式解析 HTML
    ↓
下载封面图片
    ↓
更新数据库
    ↓
回调通知 UI 更新
```

## 九、注意事项

1. 删除书籍记录只会删除数据库中的信息，不会删除源文件
2. 封面图片会自动保存到 `covers` 目录
3. 数据库文件 `ebook_manager.db` 会在程序首次运行时自动创建
4. MOBI 和 AZW3 格式的元数据解析支持有限，主要依赖文件名
5. 豆瓣解析需要配置有效的 Cookie，且有访问频率限制
6. 建议定期备份数据库文件和封面目录
